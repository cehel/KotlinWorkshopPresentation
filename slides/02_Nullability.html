Nullability

In Kotlin we differentiate between references that can be null and those who can not

Compile time error: 

                    var a : Int = 1
                    a = null
No compilation error: 

                    var a : Int?  = 1
                    a = null

Nullability Handling

Dealing with null

Variant 1: check for null


                    val a : Int? = 2       
                    val c = if(a!= null) a else -1
                

                    val d = a ?: -1

Variant 2: Call safe operator


                    val a : String? = "Hello"
                    val b = a.length
                

                    val a : String? = "Hello"
                    val b = a?.length

Variant 3: Safe-call operator in combination with let


                    val list = listOf("I", "like", "kotlin", null )
                    for(word in list){
                        word?.let { 
                            print(word.length)//smart-cast
                    }}

It will print: 146

Safe call operator is also chainable. Show in console. inside the let block, the compiler ‘smart-casts’ word to a non-nullable, so ’.length’ can be called

Variant 4: For all NPE-Lovers, the !! operator


                    var b : String? = null
                    println("${b!!.length}")
                    // it will throw a NullPointerException
When can you use "!!" ?

When can !! be used In some cases smart-cast is not working Java interoperation: You are using a Java library and it is documented that this will never return null.

Variant 5: Do it yourself


                    fun <T> T?.or(default: T): T = if (this == null) default else this
                

                    var variable: String? = "Hello"
                    println(variable?.length.or(-1))
